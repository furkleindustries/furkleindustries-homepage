fuSoundCollection :Object.<string, number>

An object containing all FuSound objects created in the current session. The keys of this object are identical to the name of the corresponding FuSound Value. For example, a successfully created FuSound with the name 'sound' will be automatically assigned to fuSoundCollection['sound']/fuSoundCollection.sound. However, there is never a need to access fuSoundCollection; all public fuSound methods will use valid FuSound names to access the relevant FuSound within the collection.

<constant> fuSoundEasings :string

An immutable enum-like object containing the possible values for easings used to fade sounds. It is impossible to change anything within this object or add to it. In ES6, attempting to do so will produce a log message; changes made in earlier versions of ES will fail silently.

fuCrossFade(outSoundObj, inSoundObj, which) → {undefined}

Stops a playing FuSound and plays a stopped or paused FuSound, fading the two in/out in an identical fashion. The FuSound being faded out must be playing, and the FuSound being faded in must be paused, otherwise the crossfade will not be performed. By default, the specific manner in which the fades are performed on both FuSounds are taken from the sound being faded out and applied to both. However, if the which argument is either "in" or equal to the FuSound being faded in, the fade information will be taken from the FuSound being faded in.
Parameters:
Name Type Argument Description
outSoundObj string | FuSound The sound to be stopped and faded out.
inSoundObj string | FuSound The sound to be played and faded in.
which string | FuSound <optional>
 If the value of which is "in," or the value of which is equal to the inSoundObj argument, inSoundObj.fade will be used to determine how the fade is performed, rather than outSoundObj.fade.

fuMakeSound(name, src, autoplay, loop, volume, fade) → {FuSound|undefined}

The factory method for creating FuSound objects. If either of the mandatory arguments (name and src) are invalid, the method will return undefined.
Parameters:
Name Type Argument Default Description
name string The name of the sound object. This corresponds exactly to the string:FuSound key-value pair within window.fuSoundCollection.
src string The source of the audio file to be played by the object.
autoplay boolean <optional>
 false Whether the sound should begin playing as soon as the object is constructed.
loop boolean <optional>
 false Whether the sound should loop back to the beginning and continue playing after the end of the track is reached.
volume string | number <optional>
 1 The volume to be assigned to the sound object. This is multiplied by the master volume level to determine the real volume of the element.
fade boolean | string | object <optional>
 The precise manner in which the sound fades in and out. How FuSound.fade is set depends on the type as the fade argument. If fade is a boolean, fade.isFading takes on that value. If fade is a string, and the string is the same one of the types of easings or timings, then fade.easing or fade.timing are set, respectively. If fade is an object, the properties corresponding with FuSound.fade's properties are assigned to FuSound.fade if their values are valid.
Properties
Name Type Argument Default Description
isFading boolean <optional>
 false Determines whether or not the volume is faded when an action (play, pause, stop, individual FuSound volume adjustment) is performed.
easing string <optional>
 'linear' Determines which of the easing functions (fuSoundEasings) to use.
timing string <optional>
 'easeInOut' Determines which of the timing types (fuSoundTimings) to use. Every easing save for linear has different effects based on the selected timing.
duration number <optional>
 1000 Determines how long the fade will last, from start to end. This value represents the number of milliseconds that will elapse.
steps number <optional>
 100 Determines the number of steps the easing function will take to fade the volume from start to finish.

fuSetMasterVolume(value) → {undefined}

Sets the master volume level, affecting all FuSound objects. When volume calculations are performed, the volume of each individual FuSound object is multiplied by this value to produce their real volume. The value must be between [0, 1]; setting the value outside that range will likely in errors each time a FuSound's volume is set or faded.
